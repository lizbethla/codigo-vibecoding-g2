Bugs detectados y corregidos durante el testing

3 bugs encontrados — todos corregidos antes de continuar al siguiente módulo
Bug #1 · Todos los ViewSets ×8
POST 201 sin campo id en respuesta
El ViewSet usaba WriteSerializer para serializar la respuesta de creación, que excluye el campo id. Detectado en el módulo customers y propagado al resto.

Fix: Override de create() en los 8 ViewSets para guardar con WriteSerializer pero responder con DetailSerializer.
✓ Corregido en views.py × 8
Bug #2 · Routes
IntegrityError no capturado en RouteStop
Violar unique_together('route', 'order') retornaba HTTP 500. DRF no puede validar este constraint a nivel de serializer porque route no es campo del WriteSerializer (se inyecta desde la URL).

Fix: Bloque try/except IntegrityError → ValidationError con HTTP 400.
✓ Corregido en routes/views.py
Bug #3 · Shipments
IntegrityError + UnorderedObjectListWarning
Mismo patrón que Bug #2 en ShipmentProductViewSet para unique_together('shipment', 'product'). Además, ShipmentProduct.Meta sin ordering generaba advertencia en paginación.

Fix: try/except + migración 0002_alter_shipmentproduct_options.
✓ Corregido en shipments/views.py + migración
